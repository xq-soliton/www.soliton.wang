ElasticSearch_单机_Install

发表于 2018-12-17 | 分类于 ElasticSearch

什么是ElasticSearch,根据其中文文档可以简单定义为以下三点,与传统的关系型数据库还是有较大的区别。
一个分布式的实施文档存储,每个字段可以被索引与搜索。
一个分布式实时分析搜索引擎。
可以胜任上百个服务节点的扩展，并支持PB级别的结构化或者非结构化数据。

系统环境


关闭防火墙 && SELINUX


ELK版本信息:


安装目录:


下载ElasticSearch安装包


Elasticsearch安装及配置：
tar zxvf /elk/elasticsearch-6.5.3.tar.gz
cd /elk/elasticsearch-6.5.3

修改elasticsearch配置文件,默认全是注释的


创建存放数据和日志的目录

mkdir -p /data/es-data
mkdir -p /logs/es

修改ES内存参数配置文件

vim /elk/elasticsearch-6.5.3/config/jvm.options

-Xms2g //开启，代表内存。所以还要根据你的需要设置
-Xmx2g //开启

ES内存分配设置

vim /etc/profile

export ES_HEAP_SIZE=2g #我虚拟机3G内存我给了2G,启动的时候回读取这个环境变量

配置linux资源限制
-在启动的时候如果没有配置这个它会报:
bound or publishing to a non-loopback address, enforcing bootstrap checks
ERROR: [2] bootstrap checks failed
[1]: max file descriptors [65535] for elasticsearch process is too low, increase to at least [65536]
[2]: max virtual memory areas vm.max_map_count [65530] is too low, increase to at least [262144]


vim /etc/security/limits.conf

soft nofile 65536 hard nofile 65536

vim /etc/sysctl.conf

vm.max_map_count=655360

执行 sysctl -p 生效,或是重启


创建elk用户

#以root用户来创建新的用户 ， groupadd 添加一个用户组
groupadd elasticsearch

#添加一个用户，-g是在用户组下 -p是密码
useradd elasticsearch -g elasticsearch -p elasticsearch

赋予目录所有者

chown -R elasticsearch.elasticsearch /elk
chown -R elasticsearch.elasticsearch /data-es-data
chown -R elasticsearch.elasticsearc /logs/es

用elasticsearch用户启动ElasticSearch

su elasticsearch

/elk/elasticsearch-6.5.3/bin/elasticsearch >> /elk/elk.log

另开终端,查看启动过程

tail -f /elk/elk.log

访问 http://IP:9200 即可访问

配置ElasticSearch启动脚本,实现自启动

如果java是用编译安装的在脚本开头记得加环境变量

export JAVA_HOME=/usr/java/jdk1.8.0_112
export JAVA_BIN=/usr/java/jdk1.8.0_112/bin
export PATH=$PATH:$JAVA_HOME/bin
export CLASSPATH=.:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar
export JAVA_HOME JAVA_BIN PATH CLASSPATH

脚本位置:/etc/init.d/
vim /etc/init.d/elasticsearch

#!/bin/sh
#chkconfig: 2345 80 05
#description: elasticsearch

case “$1” in
start)
su elasticsearch<<elk
cd /elk/elasticsearch-6.5.3/
./bin/elasticsearch -d
elk
echo “elasticsearch startup”
;;
stop)
es_pid=ps aux|grep elasticsearch | grep -v 'grep elasticsearch' | awk '{print $2}'
kill -9 $es_pid
echo “elasticsearch stopped”
;;
restart)
es_pid=ps aux|grep elasticsearch | grep -v 'grep elasticsearch' | awk '{print $2}'
kill -9 $es_pid
echo “elasticsearch stopped”
su elasticsearch<<elk
cd /elk/elasticsearch-6.5.3/
./bin/elasticsearch -d
elk
echo “elasticsearch startup”
;;
*)
echo “start|stop|restart”
;;
esac
exit $?

配置自启动

chkconfig –add elasticsearch

systemctl start elasticsearch.service //启动
systemctl sotp elasticsearch.service //关闭
systemctl status elasticsearch.service //状态
systemctl enbale elasticsearch.service //开机启动
systemctl disable elasticsearch.service //开机不启动

添加安装访问；


找到了一个利用插件进行网页访问控制,感觉还不错(主要开源).
SearchGuard
单机版的ES做到这里就结束了,后面还会做集群。
