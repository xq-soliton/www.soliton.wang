ElasticSearch_Cluster_Install

2018-12-18 ElasticSearch ELK 0 评论 字数统计: 964(字) 阅读时长: 5(分)

在前面,我已经写了ES的单机,在本文中我们来搭建集群并配置好集群,配置elasticsearch-head插件.

热心提示:3台按照单机版安装好并且进行测试访问。

主机名 or IP地址


在这里我把test_a配置为主(master)


配置集群自动发现


3台服务器配置head插件




配置文件解释:
cluster.name: soliton-es-cluster //集群名,在集群内的所有主机配置保持相同
node.name: test_a //节点名,在集群内单个节点的唯一标识
path.data: /data/es-data //ES数据存放目录
path.logs: /logs/es //ES日志存放目录
network.host: 0.0.0.0 //可访问主机
http.port: 9200 //服务端口,默认9200
transport.tcp.port: 2000 //通信端口,默认端口9300
discovery.zen.ping.unicast.hosts: [“192.168.0.16:2000”,”192.168.0.17:2000”,”192.168.0.18:2000”] //集群节点IP或是主机名,这里我可以写成test_a:2000
discovery.zen.minimum_master_nodes: 3 //这个参数控制的是，一个节点需要看到的具有master节点资格的最小数量，然后才能在集群中做操作。
http.cors.enabled: true //本行以及下一行是head插件配置
http.cors.allow-origin: “*”

3台主机启动ES服务并查看


在上一篇文章中我们有写脚本启动,那么我们直接可用系统管理启动服务。


配置ES-head插件在ES中安装使用


(test_a安装即可)
安装nodejs环境并配置nodejs全局变量


安装grunt工具及插件：


安装elasticsearch-head插件及配置。


修改elasticsearch-head下的Guntfile.js文件,默认监听本机的9200端口。


然后接着进入head插件目录执行grunt server启动
cd /elk/elasticsearch-head/ && grunt server&
访问http://test_a:9100即可

在这里我唠叨一下,说说为什么在本文中ES通信端口为什么不用默认的.


答:因为我配置通信端口是默认的情况下,ES启动可能会找不到主机路由,你们可以试试,也许会成功呢.
